The “first token” is not the first letter. It's the first chunk the tokenizer produces. GPT-2 doesn't prepend a special start marker, so token 0 is simply the first real piece of your prompt. Models like Llama or Gemma add a special <bos> token in front — a difference this paper cares about.
02 · ATTENTION
Each token decides whom to listen to
Attention is the only way tokens share information. At every layer, each token (a row) spreads 100% of its “listening budget” across the tokens before it (the columns). Brighter = more attention. Try your own prompt — then look at the first column.

04 · WHY
Why would a model waste attention on purpose?
Attention is the only channel tokens have for mixing information. Stack 12, 48, 126 layers and feed in a long context, and tokens mix with each other over and over. Past a point, every token's representation starts to look like every other token's — researchers call it over-mixing (or rank collapse). The signal turns to mush.
mo.callout(kind="info") · the classroom analogy
Imagine a classroom where, every few minutes, everyone copies a bit from their neighbors' notes. A little copying fills gaps. Too many rounds, and every notebook becomes the same blurry soup. The model's trick: when a head has nothing useful to copy, it doodles in the margin instead — it parks its attention on token 0, a near-empty spot where attending is almost a no-op. The sink is a release valve against over-mixing. That's the paper's answer.

06 · EXPERIMENT
What happens if we break the first token?
Don't take the paper's word for it — generate your own evidence. We run the model on seven prompt variants and compare how much attention the first token soaks up (deep layers, all heads). If the sink is about position, swapping the first token for nonsense or punctuation should barely matter. If it's about over-mixing pressure, longer prompts should sink harder.

Longer prompt: more mixing pressure, stronger sink — just as the theory predicts.
Heavy repetition: heads have real copying work to do, so less attention is parked on the sink.
Very short prompt: little mixing to defend against — the release valve stays mostly closed.

09 · SO WHAT
The sink is not a bug. It's a defense.
01
Sinks are learned, useful structure. Attention parked on token 0 is a near no-op that keeps token representations distinct as depth and context grow.
02
It's about position, mostly. Replace the first token with nonsense and the sink stays. But training choices (a fixed <bos>) matter too — remove one a model relied on and quality drops.
03
“Boring” patterns repay attention. A behavior people tried to delete turned out to be load-bearing. That's why interpretability matters.
